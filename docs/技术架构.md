## 一、产品定位

**链易配** 是一个面向县域经济的高质量发展平台，聚焦产业链供需智能匹配与供应链风险预警。核心价值主张：

1. **企业侧**：帮企业找到最优供应商/客户，降低采购成本
2. **政府侧**：实时监测产业链薄弱环节，提供补链强链决策支持
3. **技术侧**：大模型 + 知识图谱 + 时序预测 + GNN 图神经网络的多技术融合

## 四、核心功能详解

### 4.1 智能匹配引擎 (9维度加权评分)

供应商匹配采用多维度加权评分算法，总分由以下维度加权求和：

| # | 维度 | 权重 | 数据来源 | 技术实现 |
|---|------|------|----------|----------|
| 1 | 产品匹配 | 22% | MySQL products表 | 关键词 + 类别匹配 |
| 2 | 绿色优先 | 15% | 质量标签系统 | 碳排放等级(C/D级) |
| 3 | 地理距离 | 13% | 企业地址 + 高德API | 球面距离 + 分段衰减 |
| 4 | 历史合作 | 12% | transactions表 | 历史交易关系加分 |
| 5 | 产能匹配 | 10% | 产能日历系统 | 供应商产能vs需求量 |
| 6 | 语义相似 | 8% | Ollama LLM | 自然语言意图理解 |
| 7 | 信用评分 | 8% | 信用引擎 | 80+维度综合信用分 |
| 8 | 技术能力 | 7% | 专利/研发数据 | 技术创新能力评估 |
| 9 | GNN评分 | 5% | HAN-BPR模型 | 图结构学习嵌入 |

**距离衰减函数**（地理维度核心逻辑）：
- ≤50km → 满分100
- 50-300km → 线性衰减 100→80
- 300-1000km → 衰减 80→40
- >1000km → 衰减 40→0

**绿色优先模式**：当开启绿色供应链时，绿色维度权重从 15% 提升到 45%，适合双碳政策场景。

**Bandit自适应**：通过 Thompson Sampling 算法动态探索最优权重组合，基于历史反馈自动优化匹配质量。

### 4.2 产业链风险预警 (4维度 × 7关键产品)

对 7 种战略关键产品进行多维度风险监测：

| 级别 | 维度 | 触发条件 | 算法 |
|------|------|----------|------|
| 🔴 红色 | 进口依赖 | 进口来源国单一且占比 > 60% | `check_import_dependency()` |
| 🟠 橙色 | 跨省依赖 | 省外采购占比 > 70% | `check_interprovincial_dependency()` |
| 🟡 黄色 | 本地短缺 | 本地供应商 < 3 家 | `check_local_supplier_count()` |
| 🟠 橙色 | 绿色风险 | ≥50%供应商高碳排放 | `check_green_risk()` |

**7 种关键产品**: 芯片、电机、特种钢材、电路板、高端传感器、光刻机、精密轴承

### 4.3 LLM智能问答 (Text-to-SQL)

基于本地 Ollama (qwen2:7b) 实现自然语言查询数据库：

**工作流程**:
1. 用户输入自然语言问题（例："有多少家企业在四川？"）
2. System Prompt 注入完整数据库 Schema（7张表，全部列名和示例）
3. LLM 生成结构化 JSON（`{"type":"sql","query":"SELECT ..."}` 或 Cypher）
4. 安全校验：SQL仅允许 SELECT，Cypher 仅允许 MATCH+RETURN
5. 执行查询并格式化返回结果

**安全机制**：
- SQL 白名单：禁止 UPDATE/DELETE/INSERT/DROP/ALTER
- Cypher 白名单：禁止 CREATE/MERGE/DELETE/CALL/SET/REMOVE
- JSON 解析容错：支持 Markdown 代码块包裹、前后噪声文本

### 4.4 产业链知识图谱

**Neo4j 图结构**：
- 节点：Product (产品实体)
- 关系：SUPPLIES_TO (供应链上下游)
- PageRank 算法识别产业链关键节点
- Louvain 社区发现划分产业集群

### 4.5 时序预测

基于 statsmodels Holt-Winters 指数平滑：
- 聚合历史月度供需数据
- 预测未来 6 个月趋势
- 数据不足时回退到简单移动平均

### 4.6 政府大屏

- 关键指标仪表盘 (企业数/供需量/交易额)
- 产业链图谱可视化 (ECharts)
- 风险预警实时监控
- 供需趋势折线图
- 产业分布地图

## 六、部署架构 (macOS + Windows 双平台)

```
[macOS]                          [Windows]
├── Homebrew MySQL 9.6            ├── MySQL 8.0 (安装包)
├── Homebrew Ollama 0.24          ├── Ollama (ollama.com下载)
├── Python 3.13 (Framework)       ├── Python 3.9+ (python.org)
├── Node.js 24 (Homebrew)         ├── Node.js 20+ (安装包)
├── Neo4j 需手动安装               ├── Neo4j Desktop
└── 端口注意: 5000 被AirPlay占用   └── 端口5000通常可用
     改用端口5050
```

**跨平台兼容关键点**：
1. `.env` 文件：数据库连接字符串使用 `localhost`（非 `127.0.0.1`）
2. 虚拟环境：各自创建 `venv/`，已在 `.gitignore`
3. 前端构建：`npm run build` 产物输出到 `app/static/frontend/`
4. macOS 5000端口被 AirPlay Receiver 占用，修改 `run.py` 使用端口 5050
5. 路径分隔符：Python 使用 `pathlib.Path` 自动处理

## 七、面向竞赛的优化方案

### 🥇 优先级一：答辩展示优化 (投入最小，效果最大)

| 优化项 | 方案 | 预期效果 |
|--------|------|----------|
| **数据充实** | 导入广东省真实产业数据（`import_guangdong_data.py`），丰富到100+企业 | 演示效果从"玩具"变为"可落地" |
| **预警演示** | 预置3-5条各等级预警数据，展示完整预警闭环 | 评委可直观看到"红色→橙色→黄色"三级联动 |
| **大屏优化** | 政府大屏增加实时数据刷新动画、预警数量呼吸灯效果 | 提升视觉表现力 |
| **流程串联** | 准备演示剧本：注册→发布供需→智能匹配→下单→预警触发→政府干预 | 完整业务流程展示 |

### 🥈 优先级二：技术亮点强化

| 优化项 | 方案 | 技术价值 |
|--------|------|----------|
| **GNN落地** | 将 `gnn_model.py` 训练结果真正接入匹配引擎（当前权重仅5%） | 体现图神经网络实际应用 |
| **联邦学习场景** | 补充 `fedlab_demo.py` 为"跨企业联合训练匹配模型" | 体现隐私计算创新 |
| **LLM增强** | 为智能问答增加流式输出（SSE），当前已在 `/api/chat` 实现 | 提升交互体验 |
| **CLIP演示** | 预置几张产品宣传图，演示"拍照找供应商" | 多模态AI亮点 |

### 🥉 优先级三：工程质量提升

| 优化项 | 方案 |
|--------|------|
| **测试覆盖** | 为核心匹配算法、预警引擎补充 pytest 测试（当前覆盖率不足） |
| **API文档** | 使用 Flask-RESTX 或手动输出 API 文档，方便评委查阅 |
| **Docker化** | 编写 `docker-compose.yml` 一键部署（MySQL + Neo4j + App），评委可复现 |
| **前端优化** | 构建产物超过500KB，使用代码分割降低首屏加载时间 |

### 📊 竞赛答辩建议

1. **开场30秒定调**：直接展示政府大屏预警界面 → "我们解决的问题是产业链安全"
2. **技术2分钟**：匹配引擎→LLM问答→知识图谱算法（PageRank/社区发现）
3. **创新点突出**：
   - 9维度匹配（传统竞品通常是3-5维）
   - 本地LLM部署（不依赖外部API，展示数据安全）
   - GNN + Bandit 强化学习（学术创新）
4. **可落地性**：展示真实产业数据导入结果
5. **备选方案**：准备一个"系统宕机"的应急预案（核心功能离线可用）
