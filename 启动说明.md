# 链易配 - 启动说明 (Windows / macOS 双平台)

## 一、环境要求

| 组件 | Windows | macOS (Apple Silicon) |
|------|---------|----------------------|
| Python | 3.9+ (python.org) | 3.9+ (brew/python.org) |
| MySQL | 8.0+ (安装包) | 8.0+ (`brew install mysql`) |
| Node.js | 20+ (nodejs.org) | 20+ (`brew install node`) |
| Ollama | [ollama.com](https://ollama.com) | `brew install ollama` |
| Neo4j | Desktop版 (可选) | 待安装 (可选，图谱功能需要) |
| Redis | 可选 | 可选 |

---

## 二、首次启动步骤

### 1. 安装依赖

```bash
# Windows PowerShell / CMD
cd 链易配项目根目录
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

# macOS Terminal
cd 链易配项目根目录
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. 配置 .env

在项目根目录创建或编辑 `.env` 文件（可复制 `.env.example`）：

```
# MySQL（必填，把 password 改成你的 MySQL 密码）
DATABASE_URL=mysql+pymysql://root:你的MySQL密码@localhost/lianyipei

# Neo4j（Neo4j Desktop 默认密码一般是 neo4j 或你设置的）
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=你的Neo4j密码

# Redis（可选）
REDIS_URL=redis://localhost:6379/0

SECRET_KEY=dev-secret-key-12345

# 智能问答（可选）
# 本机 Ollama（推荐本地开发）：LLM_MODEL 填 ollama list 中的模型名
LLM_BASE_URL=http://localhost:11434/v1
LLM_MODEL=qwen2.5
LLM_API_KEY=ollama
# 或使用远程 model_service（见 MODEL_SERVICE.md）：http://<GPU服务器IP>:5001
```

### 3. 启动 MySQL 和 Neo4j

- **MySQL**：确保服务已启动（Windows 服务 / 任务管理器）
- **Neo4j Desktop**：打开软件 → 选择数据库 → 点击 **Start**

### 4. 初始化数据库

```bash
# 创建数据库
python scripts/create_db.py

# 初始化全部数据（企业、产品、供需、产业链图谱等）
python scripts/seed_all_data.py
```

### 5. 启动应用

```bash
python run.py
```

浏览器访问：**http://localhost:5000**

## 五、常见问题

### 1. MySQL 连接失败

- 确认 MySQL 已启动
- 检查 `.env` 中 `DATABASE_URL` 的账号、密码、数据库名
- 手动创建库：`CREATE DATABASE lianyipei CHARACTER SET utf8mb4;`

### 2. Neo4j 连接失败

- 确认 Neo4j Desktop 中数据库已 **Start**
- 确认端口 7687 未被占用
- 检查 `.env` 中 `NEO4J_PASSWORD`

### 3. 端口 5000 被占用

修改 `run.py` 最后一行为：`app.run(debug=True, host='0.0.0.0', port=5001)`（或其它端口）

### 4. 数据已存在，只想启动

直接运行：

```bash
python run.py
```

### 5. 智能问答无法使用（提示未配置）

- **Ollama**：确认本机 Ollama 已运行 (`ollama serve`)，`.env` 中 `OLLAMA_BASE_URL=http://localhost:11434`，模型名与 `ollama list` 一致
- **模型未下载**：运行 `ollama pull qwen2:7b` (约4.4GB，建议开启代理)
- **bizmind 模型**：`cd ollama && ollama create bizmind -f Modelfile`（基于 qwen2:7b 创建）
- **model_service**：确认远程服务已启动，详情见 `MODEL_SERVICE.md`

### 6. macOS 端口5000被占用

macOS Ventura+ 的 AirPlay Receiver 默认占用5000端口。本项目 `run.py` 已内置自动端口检测：优先5000，备用5050。也可手动在"系统设置 → 通用 → AirDrop与接力"中关闭 AirPlay Receiver。

### 7. Git 代理配置 (Clash)

若使用 Clash 代理（端口7897），配置 Git 代理以加速 clone/push：

```bash
# HTTP/HTTPS 代理
git config --global http.proxy http://127.0.0.1:7897
git config --global https.proxy http://127.0.0.1:7897

# 仅对 GitHub 设置代理（推荐）
git config --global http.https://github.com.proxy http://127.0.0.1:7897
```

### 8. 改了前端代码（如 AI 聊天流式）但浏览器里还是旧行为 / 控制台报错

- 通过 **`http://localhost:5000`（Flask 托管）** 打开的是 **`app/static/frontend/`** 里**已打包**的 `assets/main.js`，不会自动跟 `frontend/src` 同步。
- 修改 `frontend/src` 后请在项目根目录执行：
  ```bash
  cd frontend
  npm run build
  ```
- 再**硬刷新**页面（Ctrl+F5）。若仍对 `/api/chat` 当 JSON 解析，说明浏览器还在用缓存的旧 `main.js`。
- 日常开发可：`cd frontend && npm run dev`，用 Vite 热更新调试前端。
